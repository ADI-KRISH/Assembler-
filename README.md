# Assembler

## Overview
The `Assembler` folder contains the implementation of an assembler for converting assembly language code into machine code. This project is part of a broader study of compilers and virtual machines.

## Folder Structure

- **Assmebler.iml**: IntelliJ IDEA module file for the assembler project.
- **src/**: Contains the source code and related files for the assembler.
  - **Asmcode.asm.txt**: Input assembly code file.
  - **Assembler.java**: Java implementation of the assembler.
  - **bincode.asm.txt**: Output file containing the binary code generated by the assembler.
  - **output.asm.txt**: Another output file for the assembler.
  - **SymbolTable.txt**: File containing the symbol table used by the assembler.
  - **SymbolTable.txt.txt**: Duplicate or backup of the symbol table file.
  - **whtrem.asm.txt**: File with whitespace-removed assembly code.

## Features
- Converts assembly language code into binary machine code.
- Supports symbol table generation and management.
- Handles whitespace removal for cleaner assembly code processing.

## How to Use
1. Place the input assembly code in `Asmcode.asm.txt`.
2. Run the `Assembler.java` program.
3. The output binary code will be generated in `bincode.asm.txt`.

## Requirements
- Java Development Kit (JDK) installed.
- IntelliJ IDEA or any Java IDE for running the project.

## Notes
- Ensure the input assembly code follows the expected syntax.
- The `SymbolTable.txt` file should be updated if new symbols are introduced.
